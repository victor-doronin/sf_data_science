# Дипломный проект: рекомендательная система

## Слайд 1. Постановка задачи

- Задача: рекомендовать 3 товара пользователю интернет-магазина на главной странице.
- Бизнес-цель: повысить прибыль от допродаж на 20%.
- Техническая метрика: `Precision@3`.

## Слайд 2. Данные

- `events`: события пользователей, просмотры, добавления в корзину, покупки.
- `category_tree`: дерево категорий товаров.
- `item_properties`: свойства товаров, включая `categoryid` и захешированные свойства.
- Период событий: с мая по сентябрь 2015 года.

## Слайд 3. Подход

- Используем временную валидацию, чтобы не подглядывать в будущее.
- События получают веса: `view = 1`, `addtocart = 3`, `transaction = 10`.
- Для известного пользователя рекомендуем товары из его истории с максимальным скором.
- Для неизвестного пользователя используем глобальный fallback из популярных товаров.

## Слайд 4. Эксперименты

- Популярные товары по покупкам.
- Популярные товары по просмотрам.
- Глобальная взвешенная популярность.
- Персональная история пользователя с весами событий.
- История сильных действий: корзина и покупки.
- Лучший результат: `Precision@3 = 0.006584`.

## Слайд 5. Итоговая модель

- Выбрана модель `personal_history_weighted`.
- Плюсы: простая, объяснимая, быстро работает, устойчива к холодному старту через fallback.
- Ограничение: модель базовая, дальше можно улучшать item-to-item и матричной факторизацией.
- Первое свойство товаров за пределами топ-20 по покрытию товаров: `348`.

## Слайд 6. Сервис

- API реализован на FastAPI.
- Endpoint рекомендаций: `GET /recommend/{visitorid}?k=3`.
- Endpoint состояния: `GET /health`.
- Endpoint метрик: `GET /metrics`.
- Сервис упакован в Docker-образ.
